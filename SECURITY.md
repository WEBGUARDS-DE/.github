# Security Policy

## Supported Versions
Nur die aktuelle Version wird mit Security Updates versorgt.

## Reporting a Vulnerability

**Bitte keine öffentlichen Disclosures für Security Issues!**

1. Email: security@webguards.de
2. Beschreibung der Vulnerability
3. Schritt-für-Schritt Reproduzierbarkeit
4. Potentielle Impact-Analyse

Wir werden uns innerhalb von 48h melden.

## Best Practices
- Regelmäßige Dependency Updates
- Code Reviews vor Merge
- Secure by Default
